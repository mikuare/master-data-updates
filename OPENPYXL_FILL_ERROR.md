# Openpyxl Fill Error

## What It Means

- `openpyxl` expects a valid Excel style object called `Fill`
- the uploaded `.xlsx` file contains broken or unusual cell formatting
- the workbook may open in Excel, but Python sees the file structure as malformed or non-standard

## Common Causes

- the file was generated by another system, not real Excel
- the file is damaged or partially corrupted
- the file was renamed to `.xlsx` even though it is not a proper Excel workbook
- the workbook has broken styles, themes, or formatting objects
- sometimes a heavily edited template causes this

## Possible Upload-Time Scenarios

- `system` file has invalid formatting
- `hr` file has invalid formatting
- one of the existing generated report files is corrupted and the preview is trying to read it
- the uploaded file is actually `.xls` or `.csv` but saved with `.xlsx` name

## What To Do

1. Open the uploaded file in Excel.
2. Use `Save As` and save a fresh copy as `Excel Workbook (*.xlsx)`.
3. Upload that new file again.
4. If possible, remove heavy formatting, merged cells, or template styling.
5. If the file is old format, save it cleanly again from Excel before uploading.

## Best Practical Fix

- open the file in Microsoft Excel
- go to `File > Save As > Excel Workbook (.xlsx)`
- use the new saved file for upload
